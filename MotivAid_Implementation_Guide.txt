How to use this guide
This document walks you through every single step needed to build and test an AI model that estimates blood loss from smartphone photographs. You do not need to be an AI expert to follow it. Each step is explained in plain language, with the exact commands to type and the exact code to write.

The guide is organised into five parts:
1. Environment setup — installing every tool you need on your computer
2. Data preparation — organising your real images and creating synthetic training data
3. Building and training the model — the actual AI development
4. Converting for mobile — making the model work offline on Android/iOS
5. MVP prototype — a working Python web app you can demo before app integration

TIP:  Every time you see a grey box like this one, it is a command or code you can copy and run directly. Every time you see a CHECKPOINT box, stop and verify that the step worked before moving on.

Step 2.3 — Prepare the blood analog for synthetic data
For training, you need photographs of simulated blood stains on all 5 surface types. This is done by applying a blood analog solution — a mixture that looks like blood on camera.

Blood analog recipe: Mix 250 mL of warm water, 2 tablespoons of corn syrup, and red food dye. Add dye gradually, comparing to photos of your real images on a phone screen until the colour matches.

For each surface type, photograph stains of these exact volumes:

SurfaceVolume rangeSamples neededSanitary pad5, 10, 25, 50, 100, 200, 400 mL40 samples (5 per volume)Gauze (4x4)2, 5, 10, 15, 20 mL25 samples (5 per volume)Hospital sheet10, 25, 50, 100, 200, 400 mL30 samples (5 per volume)Surgical drape10, 25, 50, 100, 200, 400, 600 mL35 samples (5 per volume)Other/mixed25, 100, 300 mL15 samples
For each sample, take 6 photographs: 2 distances (20 cm, 40 cm) x 3 lighting conditions (daylight, fluorescent, LED). Name files consistently:
pad_050ml_d20_daylight_001.jpg
pad_050ml_d40_fluorescent_001.jpg
drape_100ml_d20_led_001.jpgSave all synthetic training images in dataset/synthetic_train/.

NOTE:  Surgical drapes are often blue or green. Blood on a blue drape looks different from blood on white fabric. Photograph drapes carefully and verify your analog solution colour matches your real drape images before photographing many samples.
Step 2.4 — Generate segmentation masks
The segmentation model needs to know which pixels are blood-stained. We generate an approximate mask using colour detection, then you manually fix any obvious errors.
Create scripts/generate_masks.py:
# scripts/generate_masks.py
import cv2, os, json
import numpy as np
from PIL import Image

IMAGE_DIR = 'dataset/synthetic_train'
MASK_DIR  = 'dataset/masks'
os.makedirs(MASK_DIR, exist_ok=True)

def make_mask(img_path, surface_type='pad'):
    img_bgr = cv2.imread(img_path)
    hsv     = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2HSV)

    if surface_type == 'drape':
        # Use LAB colour space for blue/green backgrounds
        lab = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2LAB)
        a_channel = lab[:, :, 1]   # A = green to red axis
        mask = cv2.inRange(a_channel, 140, 255)
    else:
        # Standard HSV blood detection
        mask1 = cv2.inRange(hsv, (0,  60, 40), (15,  255, 255))
        mask2 = cv2.inRange(hsv, (160, 60, 40), (180, 255, 255))
        mask  = cv2.bitwise_or(mask1, mask2)

    # Clean up the mask
    kernel = np.ones((7, 7), np.uint8)
    mask   = cv2.morphologyEx(mask, cv2.MORPH_CLOSE, kernel)
    mask   = cv2.morphologyEx(mask, cv2.MORPH_OPEN,  kernel)
    return mask

for fname in os.listdir(IMAGE_DIR):
    if not fname.lower().endswith(('.jpg', '.jpeg', '.png')):
        continue
    # Infer surface type from filename
    surface = 'drape' if 'drape' in fname else 'pad'
    mask = make_mask(os.path.join(IMAGE_DIR, fname), surface)
    mask_name = os.path.splitext(fname)[0] + '_mask.png'
    cv2.imwrite(os.path.join(MASK_DIR, mask_name), mask)

print(f'Masks generated for {len(os.listdir(MASK_DIR))} images.')python scripts/generate_masks.py

# scripts/build_labels.py
import os, json, re

IMAGE_DIR = 'dataset/synthetic_train'
output    = {}

SURFACES = {'pad': 0, 'gauze': 1, 'sheet': 2, 'drape': 3, 'other': 4}

for fname in os.listdir(IMAGE_DIR):
    if not fname.lower().endswith(('.jpg', '.jpeg', '.png')):
        continue
    # Extract volume from filename e.g. pad_050ml_d20_daylight_001.jpg
    m = re.search(r'_(\d+)ml', fname)
    if not m:
        print(f'WARNING: could not parse volume from {fname}')
        continue
    volume = int(m.group(1))
    surface = 'other'
    for s in SURFACES:
        if fname.startswith(s):
            surface = s
            break
    output[fname] = {'volume_ml': volume, 'surface_type': surface}

with open('dataset/synthetic_labels.json', 'w') as f:
    json.dump(output, f, indent=2)
print(f'Labeled {len(output)} synthetic images.')python scripts/build_labels.py

Step 3.2 — Create the dataset loader (scripts/dataset.py)
This file teaches Python how to read your images and labels. Create it locally (and upload to Drive).
# scripts/dataset.py
import os, json
import numpy as np
from PIL import Image
import torch
from torch.utils.data import Dataset
import albumentations as A
from albumentations.pytorch import ToTensorV2

SURFACE_MAP = {'pad':0, 'gauze':1, 'sheet':2, 'drape':3, 'other':4}

def get_transforms(mode='train'):
    if mode == 'train':
        return A.Compose([
            A.RandomResizedCrop(256, 256, scale=(0.7, 1.0)),
            A.HorizontalFlip(p=0.5),
            A.VerticalFlip(p=0.3),
            A.Rotate(limit=25, p=0.5),
            A.RandomBrightnessContrast(
                brightness_limit=0.35, contrast_limit=0.25, p=0.8),
            A.HueSaturationValue(
                hue_shift_limit=12, sat_shift_limit=25, val_shift_limit=20, p=0.7),
            A.CLAHE(clip_limit=4.0, p=0.5),
            A.GaussianBlur(blur_limit=(1, 3), p=0.3),
            A.Normalize(mean=[0.485,0.456,0.406],
                        std=[0.229,0.224,0.225]),
            ToTensorV2()
        ])
    else:
        return A.Compose([
            A.Resize(256, 256),
            A.Normalize(mean=[0.485,0.456,0.406],
                        std=[0.229,0.224,0.225]),
            ToTensorV2()
        ])

class BloodLossDataset(Dataset):
    def __init__(self, image_dir, mask_dir, labels_file, mode='train'):
        self.image_dir   = image_dir
        self.mask_dir    = mask_dir
        self.transforms  = get_transforms(mode)
        with open(labels_file) as f:
            self.labels = json.load(f)
        self.files = [
            k for k in self.labels
            if os.path.exists(os.path.join(image_dir, k))
        ]

    def __len__(self):
        return len(self.files)

    def __getitem__(self, idx):
        fname   = self.files[idx]
        info    = self.labels[fname]
        img     = np.array(Image.open(
                  os.path.join(self.image_dir, fname)).convert('RGB'))
        mname   = os.path.splitext(fname)[0] + '_mask.png'
        mask    = np.array(Image.open(
                  os.path.join(self.mask_dir, mname)).convert('L'))
        mask    = (mask > 127).astype(np.float32)
        aug     = self.transforms(image=img, mask=mask)
        volume  = float(info['volume_ml'])
        surface = SURFACE_MAP.get(info['surface_type'], 4)
        return {
            'image':   aug['image'],
            'mask':    aug['mask'].unsqueeze(0),
            'log_vol': torch.tensor(np.log(volume + 1e-6), dtype=torch.float32),
            'surface': torch.tensor(surface, dtype=torch.long),
            'volume':  torch.tensor(volume, dtype=torch.float32),
        }

The segmentation model learns to draw a mask around the blood stain. This is the first of two models you will train.
# scripts/train_seg.py
# Run this in Google Colab
import torch
import segmentation_models_pytorch as smp
from torch.utils.data import DataLoader
from dataset import BloodLossDataset
import wandb

# -- Config --
EPOCHS      = 50
BATCH_SIZE  = 16
LR          = 1e-3
DEVICE      = 'cuda' if torch.cuda.is_available() else 'cpu'
print(f'Training on: {DEVICE}')

# -- Data --
train_ds = BloodLossDataset(
    'dataset/synthetic_train', 'dataset/masks',
    'dataset/synthetic_labels.json', mode='train')
val_ds   = BloodLossDataset(
    'dataset/real_test', 'dataset/masks',
    'labels.json', mode='val')    # validate on REAL images
train_dl = DataLoader(train_ds, batch_size=BATCH_SIZE, shuffle=True,  num_workers=2)
val_dl   = DataLoader(val_ds,   batch_size=BATCH_SIZE, shuffle=False, num_workers=2)

# -- Model --
model = smp.Unet(
    encoder_name='mobilenet_v2',
    encoder_weights='imagenet',
    in_channels=3,
    classes=1,
    activation='sigmoid',
).to(DEVICE)

# -- Loss & Optimizer --
loss_fn   = smp.losses.DiceLoss('binary') + smp.losses.SoftBCEWithLogitsLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=LR, weight_decay=1e-4)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=EPOCHS)

# -- Training loop --
best_iou = 0.0
for epoch in range(EPOCHS):
    model.train()
    total_loss = 0
    for batch in train_dl:
        imgs  = batch['image'].to(DEVICE)
        masks = batch['mask'].to(DEVICE)
        pred  = model(imgs)
        loss  = loss_fn(pred, masks)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    scheduler.step()

    # Validation
    model.eval()
    ious = []
    with torch.no_grad():
        for batch in val_dl:
            imgs  = batch['image'].to(DEVICE)
            masks = batch['mask'].to(DEVICE)
            pred  = (model(imgs) > 0.5).float()
            inter = (pred * masks).sum()
            union = (pred + masks).clamp(0,1).sum()
            ious.append((inter / (union + 1e-6)).item())
    val_iou = sum(ious) / len(ious)
    print(f'Epoch {epoch+1:02d} | Loss: {total_loss/len(train_dl):.4f} | Val IoU: {val_iou:.4f}')

    if val_iou > best_iou:
        best_iou = val_iou
        torch.save(model.state_dict(), 'models/seg_best.pt')
        print(f'  >>> Saved new best model (IoU={val_iou:.4f})')

print(f'Training complete. Best validation IoU: {best_iou:.4f}')

Step 3.4 — Train the regression model (scripts/train_reg.py)
The regression model takes the masked stain region and predicts blood volume in mL. This is the second model.
# scripts/train_reg.py
# Run this in Google Colab after train_seg.py is complete
import torch
import torch.nn as nn
import torchvision.models as models
from torch.utils.data import DataLoader
from dataset import BloodLossDataset
import numpy as np

DEVICE     = 'cuda' if torch.cuda.is_available() else 'cpu'
EPOCHS     = 80
BATCH_SIZE = 32

# -- Load segmentation model to generate masked inputs --
import segmentation_models_pytorch as smp
seg_model = smp.Unet('mobilenet_v2', encoder_weights=None,
                     in_channels=3, classes=1, activation='sigmoid').to(DEVICE)
seg_model.load_state_dict(torch.load('models/seg_best.pt'))
seg_model.eval()

# -- Regression model --
class BloodLossRegressor(nn.Module):
    def __init__(self):
        super().__init__()
        backbone = models.mobilenet_v3_small(weights='IMAGENET1K_V1')
        self.features = backbone.features
        self.avgpool  = backbone.avgpool
        # 576 from backbone + 5 surface one-hot + 3 extra features
        self.head = nn.Sequential(
            nn.Linear(576 + 5 + 3, 128),
            nn.Hardswish(), nn.Dropout(0.3),
            nn.Linear(128, 32),
            nn.Hardswish(),
            nn.Linear(32, 1)
        )
    def forward(self, img, surface_oh, extras):
        x = self.avgpool(self.features(img)).flatten(1)
        x = torch.cat([x, surface_oh, extras], dim=1)
        return self.head(x).squeeze(1)

model     = BloodLossRegressor().to(DEVICE)
optimizer = torch.optim.AdamW(model.parameters(), lr=5e-4, weight_decay=1e-4)
scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
    optimizer, patience=5, factor=0.5)
loss_fn   = nn.HuberLoss(delta=0.3)

train_ds  = BloodLossDataset('dataset/synthetic_train', 'dataset/masks',
                              'dataset/synthetic_labels.json', mode='train')
train_dl  = DataLoader(train_ds, batch_size=BATCH_SIZE, shuffle=True, num_workers=2)

best_mae  = float('inf')
for epoch in range(EPOCHS):
    model.train()
    maes = []
    for batch in train_dl:
        imgs     = batch['image'].to(DEVICE)
        log_vols = batch['log_vol'].to(DEVICE)
        surfaces = batch['surface'].to(DEVICE)
        # One-hot encode surface
        surface_oh = torch.zeros(imgs.size(0), 5, device=DEVICE)
        surface_oh.scatter_(1, surfaces.unsqueeze(1), 1.0)
        # Extra features: stain area, mean hue, mean saturation (simplified)
        extras = torch.zeros(imgs.size(0), 3, device=DEVICE)
        pred   = model(imgs, surface_oh, extras)
        loss   = loss_fn(pred, log_vols)
        optimizer.zero_grad(); loss.backward(); optimizer.step()
        # MAE in mL
        pred_ml   = torch.exp(pred.detach())
        true_ml   = batch['volume'].to(DEVICE)
        maes.extend((pred_ml - true_ml).abs().cpu().tolist())
    train_mae = np.mean(maes)
    scheduler.step(train_mae)
    print(f'Epoch {epoch+1:03d} | Train MAE: {train_mae:.1f} mL')
    if train_mae < best_mae:
        best_mae = train_mae
        torch.save(model.state_dict(), 'models/reg_best.pt')
print(f'Best MAE: {best_mae:.1f} mL')

mae  = np.mean([e['error'] for e in errors])
rmse = np.sqrt(np.mean([e['error']**2 for e in errors]))
within15 = sum(1 for e in errors if e['pct'] <= 15) / len(errors) * 100
print(f'\nMAE:  {mae:.1f} mL')
print(f'RMSE: {rmse:.1f} mL')
print(f'Within 15% error: {within15:.0f}%')
CHECKPOINT:  TARGET: MAE below 45 mL on real images. If MAE is above 60 mL, see the Troubleshooting section at the end of Part 3.
Troubleshooting high MAE
If your real image MAE is too high, work through these checks in order:
15. Check the segmentation masks on your real images first. Run the seg model alone and visualise its mask output. If the mask is completely wrong (all black, or covering the whole image), the segmentation model needs more training data or better augmentation.
16. Check whether the error is worse for a specific surface type. If drapes are consistently wrong, add more drape augmentation (colour jitter targeting blue-green backgrounds).
17. Check whether error grows with volume. If small volumes (under 50 mL) are accurate but large volumes (over 200 mL) are far off, your training data may not have enough high-volume samples.
18. If everything looks reasonable but MAE is 50–70 mL, that is acceptable for a first prototype. Real clinical usefulness begins around 30 mL MAE — you will close the gap by collecting more real images over time.

SEG_MODEL = ort.InferenceSession('models/seg_model.onnx')
REG_MODEL = ort.InferenceSession('models/reg_model.onnx')

def preprocess(img_pil, size):
    img = img_pil.resize((size, size)).convert('RGB')
    arr = np.array(img, dtype=np.float32) / 255.0
    mean = np.array([0.485, 0.456, 0.406])
    std  = np.array([0.229, 0.224, 0.225])
    arr  = (arr - mean) / std
    return arr.transpose(2, 0, 1)[np.newaxis].astype(np.float32)

def estimate(img_pil, surface_type):
    # Step 1: Segmentation
    img256  = preprocess(img_pil, 256)
    mask    = SEG_MODEL.run(None, {'image': img256})[0]
    mask_bin = (mask > 0.5).astype(np.float32)
    # Stain coverage as confidence proxy
    coverage = float(mask_bin.mean())

    # Step 2: Regression
    img224   = preprocess(img_pil, 224)
    masked   = img224 * mask_bin[:, :, :224, :224]  # rough crop+apply
    s_idx    = SURFACE_MAP.get(surface_type, 4)
    s_oh     = np.zeros((1, 5), dtype=np.float32)
    s_oh[0, s_idx] = 1.0
    extras   = np.zeros((1, 3), dtype=np.float32)
    log_ml   = REG_MODEL.run(None, {
                   'image': masked,
                   'surface_type': s_oh,
                   'extra_features': extras})[0][0]
    est_ml   = float(np.exp(log_ml))
    conf     = min(coverage * 20, 1.0)   # simple confidence heuristic
    return est_ml, conf

// lib/ai/blood-loss-camera.ts
import { loadTensorflowModel } from 'react-native-fast-tflite';

export type SurfaceType = 'pad'|'gauze'|'sheet'|'drape'|'other';

export interface CameraEstimateResult {
  estimatedMl:  number;
  confidence:   number;  // 0.0 to 1.0
  imageUri:     string;
}

const SURFACE_INDEX: Record<SurfaceType, number> =
  { pad:0, gauze:1, sheet:2, drape:3, other:4 };

let segModel: any = null;
let regModel: any = null;

async function loadModels() {
  if (!segModel) {
    segModel = await loadTensorflowModel(
      require('../../assets/models/blood_loss_seg.tflite'));
  }
  if (!regModel) {
    regModel = await loadTensorflowModel(
      require('../../assets/models/blood_loss_reg.tflite'));
  }
}

export async function estimateBloodLossFromImage(
  imageUri: string,
  surfaceType: SurfaceType = 'pad'
): Promise<CameraEstimateResult> {
  await loadModels();
  const preprocessed = await preprocessImage(imageUri, 256);
  const [segMask]     = await segModel.run([preprocessed]);
  const maskedCrop    = applyMaskAndResize(preprocessed, segMask, 224);
  const surfaceOneHot = buildSurfaceOneHot(surfaceType);
  const extras        = new Float32Array(3); // zeros
  const [logMl]       = await regModel.run([maskedCrop, surfaceOneHot, extras]);
  const estimatedMl   = Math.exp(logMl[0]);
  const maskCoverage  = computeCoverage(segMask);
  const confidence    = Math.min(maskCoverage * 20, 1.0);
  return { estimatedMl, confidence, imageUri };
}

function buildSurfaceOneHot(s: SurfaceType): Float32Array {
  const arr = new Float32Array(5);
  arr[SURFACE_INDEX[s]] = 1.0;
  return arr;
}

// implement these using expo-image-manipulator for resize/normalize
// and manual array ops for masking.

Glossary — Key Terms Explained
These are the technical terms used throughout this guide, explained in plain language.

TermWhat it meansModelA file that has been trained to recognise patterns. Like a trained inspector who has seen thousands of samples.TrainingShowing the model thousands of examples until it learns the pattern. Like studying for an exam.EpochOne complete pass through all training data. Training runs for 50–80 epochs.LossA score measuring how wrong the model is. Lower is better. Goes down as training progresses.MAE (Mean Absolute Error)The average error in mL across all test images. Your main accuracy metric.IoU (Intersection over Union)Accuracy metric for segmentation. 1.0 = perfect mask. Target: above 0.75.SegmentationDrawing a boundary around the stained region in the image (which pixels are blood vs. background).RegressionPredicting a number (mL) from the image, rather than a category.PyTorchThe Python library used to build and train the model.ONNXAn intermediate model format. Bridge between PyTorch and TFLite.TFLiteTensorFlow Lite. The format that runs models offline on Android/iOS.AugmentationArtificially modifying training images (brightness, rotation) to create variety.Virtual environment (venv)An isolated Python container for your project's libraries.Google ColabFree online environment with GPU access for running training code.FlaskA simple Python web framework used to build the MVP prototype.
